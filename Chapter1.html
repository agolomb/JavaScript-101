Values, Types, and Operators

Open up the console to see the results of the functions written in JavaScript!

Computers store billions of bits in its memory. With JavaScript we represent those bits in chunks called VALUES. Values can be integers, decimals, strings, or even functions to name a few.

Numbers

When dealing with fractional numbers like Pi, these digital numbers are approximations. They are not 100% precise like 1 or 15.

Arithmetic

We can do arithmetic with JavaScript. "+" and "*" are operators. When order of operations matters, be sure to use () in your code to perform your arithmetic as intended.

The MODULO operator %

The % operator is used when dealing with remainders. Here's some examples to see how this works

314 % 100 = 14. What happened? 300/100 = 3 and then we have the remainder of 14

144 % 12 = 0. What happened? 144/12 = 12. There's no remainder. So the result is zero!

Take a look in the console for some practice results of modulo.

Special Numbers

Infinity and negative infinity are special numbers available to us but it generally isn't very useful and will often yield NaN (also a special number). Keep in mind the NaN is a number type in JavaScript.

Strings

Strings are text that are written inbetween '', "", or ``. Keep in mind that if you went to start a new line in string (when press enter), you can only do this when using the backticks OR when can use "\n" if using '' or "" inside our string.

Backticks and strings

We can do more tricks with strings when we use the backtick. Look in the console and the JavaScript examples for how this works. ***We call these TEMPLATE LITERALS***

Unary Operators

typeof is a unary operator. The typeof unary operatory can ONLY take one argument. If it you pass it more than it'll only read the last argument.

Boolean Values

These are true and false.

Comparison Operators

This is when we use <, >, ==, ===, !=, >=, and <=

Logical Operators

This is when we use && or ||.

Ternary Operator

This is a conditional operator where if the condition is true, we pick the first value OTHERWISE we go with the second value.

Null and undefined

These are both considered to be empty. They are the absence of anything meaningful. They are considered values but don't have any information. We can treat both null and undefined interchangeably.

Type Coercion

This is an important part of JavaScript because it can cause problems. Take a look at the JavaScript and console to see how this can cause issues. It's best to use the triple equals "===" to defensively protect our program from issues that can come from JavaScript's type coercion.

Odd behavior of Logical Operators && and || AKA Short Circuit

Both && and || will try to convert the first value (left side) to a Boolean to make a decision. Then depending on what happens, it'll return the original left side value or the right side value. ***If both are falsy? The last value is returned. Look at JavaScript for demonstrations of this phenomena.

Truthy and Falsy

Besides "true" and "false" certain values are interpreted as true or false that we might not expect as humans. Like the number zero. Zero is interpreted by the machine as false, so we say that zero is falsy. Another example of a falsy is "" (an empty string).

Falsy Values
  1. false: The boolean value false.
  2. 0: The number zero.
  3. -0: Negative zero (uncommon).
  4. '' (Empty String): An empty string.
  5. null: Represents the absence of any value.
  6. undefined: Represents the absence of a defined value.
  7. NaN: Stands for "Not-a-Number" and represents an invalid arithmetic operation.
Truthy values
  1. true: The boolean value true.
  2. All non-zero numbers: Any positive or negative number, including decimals.
  3. Non-empty strings: Any string that contains at least one character.
  4. All objects: Including arrays, functions, and other objects.
  5. Arrays: Even if they are empty, they are considered truthy.
  6. Functions: Even if they don't return anything explicitly, they are truthy.
  7. Special objects like Infinity and -Infinity: They are considered truthy.
